# 기술 스택

## 개요

토론장 프로젝트에서 사용된 기술 스택과 선택 이유를 정리한 문서입니다.

---

## 모노레포 & 빌드

### Turborepo

- **버전**: 2.7.2
- **역할**: 모노레포 빌드 시스템

**선택 이유**:

- **캐싱**: 변경되지 않은 패키지는 빌드를 건너뛰어 CI/CD 속도 향상
- **병렬 실행**: 의존성 그래프를 분석해 독립적인 태스크를 동시에 실행
- **간단한 설정**: Nx 대비 설정이 단순하고 학습 곡선이 낮음
- **Vercel 통합**: Vercel 배포 시 자동으로 캐싱 및 최적화 지원

### pnpm

- **버전**: 10.26.2
- **역할**: 패키지 매니저

**선택 이유**:

- **디스크 효율성**: 하드 링크를 사용해 중복 패키지 저장 방지 (npm/yarn 대비 70% 이상 절약)
- **빠른 설치**: 병렬 설치 및 캐싱으로 npm보다 2-3배 빠름
- **엄격한 의존성**: phantom dependency 문제 방지 (명시적으로 선언하지 않은 패키지 접근 차단)
- **워크스페이스 지원**: `workspace:*` 프로토콜로 모노레포 내 패키지 간 의존성 관리 용이

---

## 프레임워크 & 라이브러리

### Next.js

- **버전**: 16.1.1
- **역할**: React 풀스택 프레임워크

**선택 이유**:

- **App Router**: 서버 컴포넌트 기본 지원으로 번들 크기 감소 및 초기 로딩 속도 향상
- **서버 사이드 렌더링(SSR)**: SEO 최적화 및 초기 페이지 로딩 성능 개선
- **API Routes**: 별도 백엔드 없이 간단한 API 엔드포인트 구현 가능
- **Turbopack**: Webpack 대비 10배 빠른 개발 서버 (Next.js 13+)
- **이미지 최적화**: `next/image`로 자동 이미지 최적화 및 lazy loading
- **파일 기반 라우팅**: 직관적인 라우팅 구조로 개발 생산성 향상

### React

- **버전**: 19.2.3
- **역할**: UI 라이브러리

**선택 이유**:

- **React 19 신기능**: Actions, useOptimistic, use() 훅 등 새로운 패턴 활용
- **서버 컴포넌트**: 클라이언트 번들에서 제외되어 성능 향상
- **Suspense 개선**: 데이터 페칭과 코드 스플리팅의 선언적 처리
- **생태계**: 풍부한 라이브러리 및 커뮤니티 지원

### TypeScript

- **버전**: 5.x
- **역할**: 정적 타입 시스템

**선택 이유**:

- **타입 안정성**: 컴파일 타임에 타입 오류 검출로 런타임 에러 방지
- **개발 경험(DX)**: 자동완성, 리팩토링, 코드 네비게이션 지원
- **문서화**: 타입 자체가 코드의 문서 역할
- **대규모 협업**: 팀 프로젝트에서 코드 품질 및 일관성 유지

## 스타일링

### Tailwind CSS

- **버전**: 4.x
- **역할**: 유틸리티 기반 CSS 프레임워크

**선택 이유**:

- **빠른 개발**: 클래스명만으로 스타일링, 별도 CSS 파일 관리 불필요
- **일관된 디자인**: spacing, color 등 디자인 토큰 기반으로 일관성 유지
- **작은 번들**: PurgeCSS로 사용하지 않는 스타일 자동 제거
- **Tailwind v4 신기능**:
  - CSS-first 설정 (`@import "tailwindcss"`)
  - `@source` 지시어로 컨텐츠 경로 지정
  - 더 빠른 빌드 성능
- **반응형 디자인**: `sm:`, `md:`, `lg:` 등 브레이크포인트 프리픽스로 간편한 반응형 구현
- **다크 모드**: `dark:` 프리픽스로 손쉬운 다크 모드 지원

## 코드 품질

### Biome

- **버전**: 2.3.10
- **역할**: 린터 + 포매터 (ESLint + Prettier 대체)

**선택 이유**:

- **올인원**: 린팅과 포매팅을 하나의 도구로 처리
- **속도**: Rust로 작성되어 ESLint + Prettier 조합 대비 10-20배 빠름
- **제로 설정**: 합리적인 기본 설정으로 별도 설정 없이 바로 사용 가능
- **일관성**: 린트 규칙과 포맷 규칙 간 충돌 없음 (ESLint + Prettier 조합의 고질적 문제 해결)
- **단일 의존성**: 여러 플러그인 설치 불필요

---

## HTTP 통신

### Axios

- **버전**: 1.13.2
- **역할**: HTTP 클라이언트

**선택 이유**:

- **인터셉터**: 요청/응답을 가로채 토큰 주입, 에러 핸들링 등 공통 로직 처리
- **자동 JSON 변환**: 요청/응답 데이터 자동 직렬화/역직렬화
- **타임아웃 설정**: 네트워크 지연에 대한 제어 가능
- **취소 토큰**: 불필요한 요청 취소로 리소스 절약
- **브라우저/Node.js 호환**: 동일한 API로 클라이언트/서버 모두에서 사용

## 프로젝트 구조

```
pilmin-fe/
├── apps/
│   ├── web/          # 메인 웹 앱 (랜딩, 토론 목록)
│   ├── admin/        # 관리자 앱
│   └── user/         # 사용자 앱 (토론 참여)
├── packages/
│   └── ui/           # 공유 UI 컴포넌트 및 아이콘
└── docs/             # 문서
```

**모노레포 구조 선택 이유**:

- **코드 공유**: `@pilmin/ui` 패키지를 통해 컴포넌트, 아이콘 재사용
- **일관된 버전 관리**: 모든 앱에서 동일한 React, TypeScript 버전 사용
- **단일 저장소**: PR 하나로 여러 앱에 걸친 변경사항 관리 가능
- **독립 배포**: 각 앱은 독립적으로 빌드 및 배포 가능
